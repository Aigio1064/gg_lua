Config help

Freeze interval:

The interval through which GameGuardian overwrites the values in the game memory.

Too small values can cause a high load on the processor and lags in the game.

Saved list updates interval:

The interval through which GameGuardian updates the values in the saved list.

Too small values can cause a high load on the processor and lags in the game.

Icons size:

The size of the icons in the GameGuardian interface.
Does not apply to a floating icon.

Ignore unknown characters:

Can help solve problems when copying numbers from different places, if special characters are copied along with the numbers.

Use notification:

Use of notifications in the system notification area of Android.
On some firmware, this increases the priority of the application and prevents it from closing.
Also you can use the notification to go to GameGuardian or the game by clicking on it.
This can be useful if you can not click on the floating icon.

Use sound effects:

System sounds when you press buttons and items.
On some firmware, sounds cause the GameGuardian to crash.

Allow suggestions from keyboard:

Hints from an external keyboard.
On some firmware, hints cause the GameGuardian to crash.

Autopause game:

Automatically puts the game on pause when opening the GameGuardian interface.

Interface acceleration:

Hardware or software interface acceleration GameGuardian.
Depending on the game and firmware, some of the options can cause problems.
If both work, use software acceleration because it consumes less memory.

This option is separate from the launch icons and only affects the display of the GameGuardian interface.
Launch icons are responsible for the type of acceleration of the start screen.

Hardware acceleration consumes much more memory, but it works more smoothly. If you have little RAM, always use software acceleration.

If hardware acceleration causes errors, GameGuardian can switch to software acceleration.

Deep read:

Reading memory by all available methods, if it could not be read.
May slow down memory readings.

System calls mode:

On some firmware, for example Lenovo, the alternative mode can work faster.

Normally, you do not need to change this option.

waitpid mode:

On some firmware, the waitpid system call does not work as expected. This is usually due to incorrect SeLinux settings.
This leads to an endless search in GameGuardian.
You can try using an alternative mode. Maybe this will work.

Fast freeze:

It works much faster than normal freezing, but not for all memory.

If the freeze does not work, disable this option.

Use visible data type in memory editor:

Do not ask for the data type in the memory editor if only one data type is displayed.

Prevent unload:

Prevent unloading GameGuardian from memory (force close).

Each subsequent level is more powerful than the previous one and includes it in itself.

The third level can cause problems (hang toast, reboots or crashes) on some firmware and emulators.

Without root only the third level works.

Bypass mode for ptrace protection:

Indicates what to do after disabling the protection connection:
    restore - restore connection
    nothing - do nothing (usually the protection itself restores the connection)
    freeze - freeze the thread (as in previous versions)

If you restore the protection, or allow the protection to do it yourself, the freezing of values in GameGuardian can cause lags due to the permanent connection / disabling of protection.

Use root in virtual space:

Using root from virtual space can help hide the fact that there is a GameGuardian on the device.

It does not work on all firmware and not on all virtual spaces.

Virtual space, firmware and root should allow working with root from virtual space.
The virtual space should allow you to install applications inward without installing them into the system.

Not all virtual spaces can provide this.

Not on all firmware this mode will work. And not with all root.

Since the root rights will be granted to the entire virtual space, you should not install anything else in it. Otherwise, any application from the virtual space can use root rights with unpredictable consequences.

The game must be installed outside the virtual space.

GameGuardian should be installed only inside the virtual space, without installing it into the system.
How to do this - see the help of a particular virtual space.

For example, for VirtualXposed, you can clone GameGuardian from the system in VirtualXposed, and then delete GameGuardian from the system.
